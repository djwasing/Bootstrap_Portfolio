Initial Blog Post / MongoDB - 3/27/19
This area of my site is going to be dedicated to blog posts. The blogs will be centered around my learning of new coding languages and technologies. I thought this would be a good way to review information and explore a bit deeper into the tech I’m currently studying. I’ve found I feel more confident if I can explain the language in my own terms to someone else, so this blog is (selfishly) a tool for my own learning.

I started the KU Coding Bootcamp in November, 2018 and have learned so much about JavaScript, CSS, Bootstrap, jQuery, MySQL, Node.js, and so many others. Obviously learning all this new information has come with challenges. There have been several times I’ve had to spend hours looking over code to find bugs, or reaching out to tutors or teachers for assistance. I think knowing how to properly utilize your resources is just as important as knowing the languages.

Something that hasn’t been a challenge is having the support of my wife Jackie. I’m so thankful to her for giving me her full support throughout the last several months. Starting this course just one and a half months after getting married I’m sure was not ideal, as she would have liked more time for us to spend together during the “honeymoon phase.” But she has stood by me every step of the way.

The latest technology we’re learning about is MongoDB. MongoDB is a document-oriented database program. It uses objects similar to JSON to navigate the database. There is a hierarchy of elements that make up the objects that starts with the “database”. Databases consist of multiple “collections.” Collections consist of multiple “documents.” And documents contain “fields.” When writing queries you simply use dot notation as you would for a JSON object to work through the collections, documents, and fields to find the data you want. Here is an example of a query for a MongoDB database: 


app.get("/all", function(req, res) {
    db.collections.find({}, function(err, data) {
      if (err) {
        console.log(err);
      }
      else {
        res.json(data);
      }
    });
  });

Currently, this code would return all the information in the database. This data will likely be returned in the order it was created, but it’s not guaranteed to be. It will really be returned in the order the data is found.

In the {} of the .find() method the user can specify which exact fields they’re looking for. The user can also sort the data in order to display it alphabetically, or chronologically from when it was created, or by several other possibilities.

In class we created a database and populated it with data about different animals. Then we wrote queries to extract and sort the data by different parameters. For example, I wrote a query to sort and show the data by the animals weight in descending order.

MongoDB has been much easier to pick up and learn thanks to my experience using JSON. The syntax made sense to me more quickly than other languages we’ve learned during the program.

That’s all for now. Happy coding!
